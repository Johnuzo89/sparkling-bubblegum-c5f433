Principles

Standards before software

Where an accepted standard exists for structuring, governing, or accessing research data, UndosaTech would adopt it rather than inventing an alternative.

Data stays where it is understood

Analysis happens inside a controlled environment researchers work in, rather than by moving data to researchers.

UK-resident by default

All data, analyses, and audit logs remain in UK-resident infrastructure. Cross-jurisdictional access requires explicit permission.

Enforced in software

Access controls, output review, and audit logging enforced by the platform itself, not policy documents.

Federated analysis, so data never moves

Where a study spans more than one institution, UndosaTech uses federated learning: the model travels to each site, trains against the data in place, and only aggregate weight updates return — optionally under differential privacy. No patient-level records are copied, pooled, or transferred at any point.

This is a proven method, not an experiment. Federated models have achieved ~99% of centralised performance in multi-institutional brain-imaging studies (Sheller et al., Scientific Reports, 2020), and a single clinical model has been trained across 20 institutions worldwide without any data sharing (Dayan et al., Nature Medicine, 2021).

For a data custodian, the practical effect is that participation does not create a new copy of the dataset anywhere — which shortens governance review, keeps the institution in control, and aligns with UK GDPR Article 89 research safeguards and the FAIR principles that UK funders now expect.
